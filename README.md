## SQL-Project

This project analyzes hotel operations data using SQL for database design and querying.

### Hotel Management System (SQL Project)
### Overview

This project focuses on building and analyzing a Hotel Management System database. It manages hotel details, rooms, guests, bookings, and payments. SQL queries are used to extract insights about hotel performance, customer trends, and pricing patterns.

### Tools Used

SQL

Relational Database Concepts

Joins, Subqueries, Window Functions

Aggregate and Filtering Queries

### Project Workflow

*Database Creation*: Designed and created the HotelDB database.

*Table Design*: Built tables for Hotels, Rooms, Guests, Bookings, and Payments with proper constraints.

*Relationship Setup*: Established relationships between tables using primary and foreign keys.

*Data Querying*: Performed multiple SQL queries to retrieve, filter, and analyze data.

### Key Insights / Query Highlights

*Guest Location Analysis*: Identified guests from cities like Mumbai, Delhi, and Pune.

*Room Price Analysis*: Filtered rooms within a specific price range (₹3,000–₹6,000).

*Payment Trends*: Found payments made using card-based methods.

*Recent Activity*: Retrieved the latest hotel bookings using sorting and limits.

*Hotel Performance* Counted number of bookings per hotel using JOIN and GROUP BY.

*Room Ranking*: Ranked rooms in each hotel based on price using DENSE_RANK().

*Comparative Pricing*: Identified hotels with room prices above the overall average using subqueries.

### Outcome

Demonstrates how SQL can manage structured data efficiently.

Shows practical use of filtering, grouping, joins, and advanced functions.

Helps in understanding business insights from hospitality data.
